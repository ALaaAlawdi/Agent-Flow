# Autonomous Improvement Log

## 2026-07-20T00:43:27Z — Task-scoped external worker capabilities

- **Selected problem:** CLI evidence submission trusted caller-supplied `--actor`, allowing an external worker to impersonate any assigned agent. This was priority backlog item 1.
- **Improvement:** Added HMAC-SHA256 bearer capabilities containing mission, task, assigned worker, nonce, issue time, and five-minute expiry. External CLI completion now reads a bounded capability from standard input instead of trusting `--actor` or exposing the bearer token in process arguments; human launch approval remains on its distinct `--actor`/rationale path. CLI worker commands require `AGENT_FLOW_CAPABILITY_SECRET` of at least 32 bytes. Reframed the product as a company staffed by AI agents, with Hermes-inspired tools, profiles, delegation, memory, skills, durable work, and gateways beneath an independent Agent-Flow control plane.
- **Test-first evidence:** Observed focused RED failures for the missing runtime API, expiry support, forged-expiry acceptance, cross-mission reuse, missing CLI token output, absent CLI secret enforcement, and weak-secret acceptance. Each behavior was followed by a focused GREEN run.
- **Files changed:** `AGENTS.md`, `agent_flow/runtime.py`, `agent_flow/cli.py`, `tests/test_runtime.py`, `tests/test_cli.py`, `README.md`, `docs/VISION.md`, `docs/ARCHITECTURE.md`, `docs/AUTONOMY_LOG.md`.
- **Verification:** Focused security/CLI tests observed RED before implementation and GREEN afterward. Final full suite: 21 tests passed with `ResourceWarning` promoted to errors. `compileall` and `git diff --check` exited 0. Added-line static scan found no literal credential, shell injection, eval/exec, pickle, or SQL-formatting issue; its sole secret-pattern match was the documented runtime secret-generation command. Independent review iterations rejected command-line bearer exposure, pre-authentication JSON parsing, unbounded stdin, non-canonical base64url, permissive claims, and incomplete negative coverage. The final implementation uses bounded stdin; positive TTLs; bounded token/payload sizes; canonical unpadded base64url; fixed signature length; HMAC verification before JSON parsing; an exact claims schema; strict nonce, issued-at, and expiry invariants; and future-issued rejection. Regression tests explicitly cover `+`, `/`, and padding rejection; parse-after-HMAC ordering; expiry/TTL/future-time bounds; wrong-worker assignment; and worker-capability rejection at the human gate. A final gate was requested on the complete staged change.
- **Known limitations:** Tokens are bearer credentials with no pre-expiry revocation or durable nonce-consumption table. Packet delivery and worker process isolation are operator responsibilities. The local CLI is a trusted gateway adapter and does not authenticate a human or process identity. The internal Python runtime method remains for trusted deterministic/demo callers and must not be exposed to workers.
